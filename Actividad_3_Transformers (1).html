# Importar bibliotecas necesarias
import pandas as pd
from transformers import BertTokenizer
import torch

import warnings

warnings.simplefilter(action='ignore', category=UserWarning)
pd.options.mode.chained_assignment = None  # default='warn'

# Cargar el conjunto de datos
file_path = 'Noticias.xlsx'
data = pd.read_excel(file_path)

# Eliminar las filas con valores faltantes
data.dropna(inplace=True)

# Preprocesar los datos
tokenizer = BertTokenizer.from_pretrained('bert-base-uncased')
data['input_ids'] = data['contenido'].apply(lambda x: tokenizer.encode(x, add_special_tokens=True))
max_len = 128
data['input_ids'] = data['input_ids'].apply(lambda x: x[:max_len] + [0]*(max_len-len(x)) if len(x) < max_len else x[:max_len])

## Etiqueta para detectar deportes y filtrar noticias de archivo
data=data[data['Etiqueta']!="archivo"]
data['Etiqueta'] = [1 if x == "deportes" else 0 for x in data["Etiqueta"]]

# Dividir el conjunto de datos en entrenamiento y validación
from sklearn.model_selection import train_test_split
X_train, X_val, y_train, y_val = train_test_split(data['input_ids'].tolist(), data['Etiqueta'].tolist(), test_size=0.2, random_state=42)

# Conversión a tensores de PyTorch
train_encodings = torch.tensor(X_train)
val_encodings = torch.tensor(X_val)

# Convertir etiquetas a tensores y ajustar su forma a (num_samples, 1).
# Se cambia reshape
train_labels = torch.tensor(y_train, dtype=torch.float32).unsqueeze(1)
val_labels = torch.tensor(y_val, dtype=torch.float32).unsqueeze(1)

## Creación de máscaras de atención
# Genera máscaras de atención (1.0 para tokens reales, 0.0 para padding)
train_masks = torch.tensor([[float(i != 0) for i in ii] for ii in X_train])
val_masks = torch.tensor([[float(i != 0) for i in ii] for ii in X_val])

Documentacion¶

  • Se realiza carga de librerias y base de datos.
  • Se realiza limpieza de los datos, eliminando filas con valores faltantes
  • Se usa la tokenizacion de BERT para convertir el texto de cada noticia en una secuencia de numeros.
  • Luego, todas las secuencias se recortan o rellenan (padding) para que tengan la misma longitud (128 en este caso). Es importante para no trabajar con secuencias de diferentes tamaños
  • Se eliminan las noticias cuya etiqueta sea "archivo", ya que solo nos interesan las categorias especializadas.
  • Despues, cambiamos la columna de etiquetas para que sea Binaria 1 ponga 1 si es "Deportes" y 0 para las demas. Asi convertimos el problema en una clasificacion binaria.
  • Separamos los datos en dos partes: entrenamiento (80%) y validacion (20%), para poder entrenar el modelo.
  • Convertimos las listas tokens y etiquetas a tensores de PyTorch, ya que BERT espera recibir los datos en este formato.
  • Por ultimo, creamos las "attention mask", que son arreglos que indican al modelo que posiciones del input son palabras reales y cuales son solo relleno (padding). Esto ayuda a BERT para que ignore el padding durante el entrenamiento.

from sklearn.metrics import f1_score, roc_auc_score, accuracy_score
from transformers import EvalPrediction
import torch
import numpy as np  # Asegúrate de importar numpy

# Desactiva WANDB para evitar logging externo.
import os
os.environ["WANDB_DISABLED"] = "true"

# fuente: https://jesusleal.io/2021/04/21/Longformer-multilabel-classification/
def multi_label_metrics(predictions, labels, threshold=0.5):

    # Aplica la función sigmoide a las predicciones (logits) para obtener probabilidades.
    sigmoid = torch.nn.Sigmoid()
    probs = sigmoid(torch.Tensor(predictions))

    # Convierte las probabilidades en predicciones binarias (0 o 1) usando el umbral.
    y_pred = np.zeros(probs.shape)
    y_pred[np.where(probs >= threshold)] = 1

    # Calcula las métricas F1-score (micro), ROC AUC (micro) y Accuracy.
    y_true = labels
    f1_micro_average = f1_score(y_true=y_true, y_pred=y_pred, average='micro')
    roc_auc = roc_auc_score(y_true, y_pred, average = 'micro')
    accuracy = accuracy_score(y_true, y_pred)

    # devuelve las métricas como un diccionario
    metrics = {'f1': f1_micro_average,
               'roc_auc': roc_auc,
               'accuracy': accuracy}
    return metrics

def compute_metrics(p: EvalPrediction):
    # Obtiene las predicciones (logits) del modelo
    preds = p.predictions[0] if isinstance(p.predictions, tuple) else p.predictions

    # Llama a la función 'multi_label_metrics' para calcular las métricas.
    result = multi_label_metrics(
        predictions=preds,
        labels=p.label_ids)
    return result

Documentacion¶

  • f1_score, roc_auc_score, accuracy_score de sklearn.metrics: Sirven para calcular métricas muy comunes en tareas de clasificación.
  • EvalPrediction de transformers: Se usa para facilitar el manejo de las predicciones y etiquetas en el entrenamiento/evaluación de modelos con Hugging Face.
  • torch: Usado para trabajar con tensores y funciones como la sigmoide.
  • Se crea un diccionario "metrics", con los resultados.

Análisis de resultados¶

1. Métricas generales¶

Al observar la tabla de métricas generales, noto que el modelo tiene un eval_loss bajo (0.179), lo que indica que el modelo cometió pocos errores en el conjunto de validación. El eval_accuracy es de aproximadamente 0.97, lo que significa que el modelo acertó el 97% de las veces al clasificar los ejemplos. Además, el eval_f1 y el eval_roc_auc también son altos (0.97), lo que sugiere que el modelo tiene un buen equilibrio entre precisión y recall, y además discrimina bien entre las dos clases.

2. Reporte de clasificación¶

En el reporte de clasificación:

  • Para la clase no_deportes la precisión es 0.99 y el recall es 0.98. Eso significa que casi todos los ejemplos de "no_deportes" son identificados correctamente, y además, casi todos los que predice como "no_deportes" realmente pertenecen a esa clase.
  • Para la clase deportes la precisión es 0.90 y el recall es 0.92. Aquí el modelo también lo hace bien, aunque es un poco menos preciso que con la otra clase, probablemente porque hay menos ejemplos de "deportes" (139 frente a 760 de "no_deportes").
  • El accuracy global es 0.97, lo que confirma que el modelo es bastante bueno clasificando ambos tipos de ejemplos.
  • El macro avg y el weighted avg también son altos (alrededor de 0.95 y 0.97 respectivamente), lo que indica que el rendimiento es muy bueno incluso cuando se promedian las métricas entre ambas clases.

3. Curva de pérdida (loss)¶

La curva de aprendizaje muestra que la pérdida disminuye de manera constante a medida que avanzan los pasos de entrenamiento. Esto es una señal positiva, ya que indica que el modelo está aprendiendo correctamente y no hay señales de sobreajuste (la pérdida no aumenta en ningún momento).

4. Conclusiones¶

  • El modelo BERT entrenado es muy efectivo para distinguir entre textos de "deportes" y "no_deportes".
  • Hay un pequeño margen de mejora en la clase "deportes", donde la precisión podría aumentar, posiblemente ajustando el balance de clases o aplicando técnicas de data augmentation.
  • El proceso de entrenamiento fue exitoso, ya que la pérdida bajó de forma constante y las métricas de desempeño son altas en todos los aspectos.
