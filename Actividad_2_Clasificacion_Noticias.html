Ejercicio 3: Preprocesamiento de Texto¶

Objetivo: Preparar los datos textuales para el entrenamiento del modelo.

Acciones:

  • Convertir todo el texto a minúsculas para uniformidad.
  • Eliminar puntuación y números para reducir el ruido en los datos.
  • Eliminar palabras comunes (stop words) que no aportan significado específico al análisis.
  • Tokenizar el texto, es decir, dividir el texto en palabras individuales.

pip install keras tensorflow

DOCUMENTACION¶

En esta etapa, preparamos los datos textuales para el entrenamiento de los modelos de PLN. El preprocesamiento es esencial para reducir el ruido y estandarizar la información, facilitando que los modelos aprendan patrones relevantes.

Pasos realizados:

  1. Conversión a minúsculas: Se transforma todo el texto a minúsculas para evitar que palabras iguales sean tratadas como diferentes por las mayúsculas/minúsculas.
  2. Eliminación de puntuación y números: Se eliminan todos los signos de puntuación y los dígitos numéricos, reduciendo así el ruido en los datos.
  3. Tokenización: Se divide el texto en palabras individuales (tokens) usando el tokenizador de NLTK.
  4. Eliminación de stop words: Se eliminan las palabras más comunes del español que no aportan significado relevante (stopwords), usando la lista de NLTK.
  5. Resultado: El texto limpio y preprocesado se almacena en una nueva columna llamada contenido_preprocesado.

Nota: Este preprocesamiento es necesario para que los modelos RNN y LSTM trabajen con datos consistentes y enfocados en el significado central de cada noticia.

import nltk
import string
from nltk.tokenize import word_tokenize
from nltk.corpus import stopwords

import keras_nlp
import tensorflow as tf

# Descargar recursos necesarios de NLTK (solo la primera vez)
nltk.download('punkt')
nltk.download('stopwords')
nltk.download('punkt_tab')

# Función de preprocesamiento de texto
def preprocess_text(text):
    # 1. Convertir texto a minúsculas
    text = text.lower()
    # 2. Eliminar signos de puntuación y dígitos
    text = ''.join([c for c in text if c not in string.punctuation and not c.isdigit()])
    # 3. Tokenizar el texto (dividir en palabras)
    tokens = word_tokenize(text)
    # 4. Eliminar stopwords (palabras vacías) en español
    tokens = [word for word in tokens if word not in stopwords.words('spanish')]
    return ' '.join(tokens)

# Aplicar el preprocesamiento a la columna 'contenido'
data_filtrada['contenido_preprocesado'] = data_filtrada['contenido'].apply(preprocess_text)

3.1 Tokenización y Padding¶

En este bloque, convertimos el texto preprocesado en secuencias numéricas que puedan ser utilizadas por los modelos de deep learning.

Pasos realizados:

  1. Tokenización:
    Se utiliza el Tokenizer de Keras para asignar un número único a cada palabra del corpus preprocesado.

  2. Conversión a secuencias:
    Cada texto se transforma en una lista de números (secuencias), donde cada número representa una palabra.

  3. Padding:
    Las secuencias se rellenan (padding) para que todas tengan la misma longitud (max_len), facilitando su uso en modelos como RNN o LSTM.

  4. Visualización:
    Se muestran las primeras cinco secuencias tras el preprocesamiento y padding.

Nota: También se puede experimentar con técnicas de embedding o preprocesamientos más avanzados, según lo realizado en ejercicios anteriores.

padded_sequences = pad_sequences(sequences, maxlen=max_len, padding='post')

from keras.models import Sequential
from keras.layers import Embedding, SimpleRNN, Dense
import matplotlib.pyplot as plt

# --- Construcción del modelo RNN ---
model_rnn = Sequential()
model_rnn.add(Embedding(input_dim=len(word_index) + 1, output_dim=128, input_length=max_len))
model_rnn.add(SimpleRNN(units=64, return_sequences=False))
model_rnn.add(Dense(units=len(categorias_relevantes), activation='softmax'))

# --- Compilación del modelo ---
model_rnn.compile(optimizer='adam', loss='sparse_categorical_crossentropy', metrics=['accuracy'])

# --- Entrenamiento del modelo ---
history_rnn = model_rnn.fit(X_train, y_train, validation_data=(X_val, y_val), epochs=10, batch_size=32)

# --- Visualización de resultados ---
plt.figure(figsize=(10, 6))
plt.plot(history_rnn.history['accuracy'], label='Entrenamiento')

Conclusiones Principales¶

  1. Desempeño comparativo

    • El modelo RNN obtuvo un rendimiento general superior al LSTM en esta tarea de clasificación de noticias, con una precisión del 71% frente al 39% del LSTM.
    • El modelo RNN logró métricas equilibradas en la mayoría de las categorías, con un desempeño especialmente destacado en la clase "cultura".
    • El modelo LSTM, en cambio, mostró un comportamiento desbalanceado, clasificando casi todas las noticias como "cultura" y fallando en identificar correctamente las demás categorías.

  2. Capacidad de generalización

    • El RNN demostró una mejor capacidad para generalizar a diferentes clases, mientras que el LSTM mostró problemas de sobreajuste o falta de convergencia, posiblemente debido a una configuración subóptima o insuficiencia de datos.

Recomendaciones para Mejorar el Rendimiento¶

  1. Ajustar la arquitectura y los hiperparámetros

    • LSTM: Probar con mayor número de unidades, capas adicionales, regularización con Dropout, y aumentar el número de épocas de entrenamiento.
    • Ambos modelos: Experimentar con técnicas de optimización como el ajuste del learning rate y el uso de callbacks para early stopping.

  2. Balance de clases

    • Si existe un desbalance en el número de ejemplos por clase, aplicar técnicas de balanceo como sobremuestreo (oversampling) de clases minoritarias o submuestreo (undersampling) de clases mayoritarias.
    • Usar métricas adicionales (F1-score, recall) para evaluar el desempeño en todas las clases y no solo la precisión global.

  3. Preprocesamiento y representación de texto

    • Explorar representaciones de texto más ricas, como Word2Vec, GloVe, o embeddings preentrenados como BERT.
    • Revisar y mejorar el preprocesamiento: limpieza de texto, eliminación de stopwords, etc.

  4. Explorar modelos alternativos

    • Probar arquitecturas más avanzadas como GRU, bidireccionales (Bidirectional LSTM/GRU), o modelos basados en Transformers (por ejemplo, BERT, DistilBERT).
    • Implementar modelos ensemble combinando varias arquitecturas para mejorar la robustez.
